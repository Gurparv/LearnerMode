### Infosys (Credit card Processing System)
**Situation** - It was a credit card dispense system for individuals and businesses. The Individual form was about 5-6 pages whereas the business form was 15-16 pages long. This Front end was connected to a Loan processing system, both worked together to process customer's application for credit card.

**Challenge** - It was a manual process to fill the forms and a lot of times because of third party integration we were not able to process the application successfully. 

**Act** - I took the initiative to learn automation of this task using Selenium and then implemented automation form filling for both of them. 

**Result** - It helped the team to bypass the repetitive work and added a jump in productivity.

## Grazitti (Storage SKUs)

**Situation** - The project was about moving our storage services which were based on pay-per-GB model to pay-per-subscription model. So earlier customer used to pay for 100GB of data, with new feature they will be paying per month but for unlimited storage.

**Challenge** - This project was important but since devs were working on other projects as well. Not a lot of serious attention was given to it. When i joined (i was the only QA) there were only 5 tickets with very low details and no BRD

**Act** - So instead of focussing on those docs and cause a bottleneck. I switched to Action model, and treated it like a startup thing. I started taking calls with different stakeholders to get sense of what their idea of the product will be and how people will be using it. 

Result - By the time, the tickets were added with details and docs were created like BRD etc. The project's testing cause any bottleneck and the project was released successfully. When i looked back at the jira board i realised that i had created 55 bugs in that.. so total tickets went from 5 to 60 when the project ends. I was appreciated by my manager for handling it so wel..

## Grazitti (Business Systems e2e flow)

**Situation** -> I was solo QA for this whole e2e flow which starts from Lead creation then moves to Account creation, further to Contact creation, on whose name the quote used to be created with products and then contract was sent to client , and once the contract was signed then order was placed. With their details sent to multiple downstream systems like Billing Manager and SMP which used to manage the user licenses. 

Challenge - As the flow was huge and in QA environments there wasnt a lot of stability due to 3rd party integrations and dependencies. Testing via old approach that depends on stable environment was not possible.

Act - I came up with efficient ways to test it so that we are not blocked due to environment. I broke down the flows into small parts and proceed with the testing. 

Result - The project went to production without any bugs and in tight deadlines.
