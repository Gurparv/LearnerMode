## What is Jenkins
Free and open source automation server. It is like a robot-assistant which will automatically do specific tasks on the computer related to building, testing and deploying a software.

So in real world use case - Everytime a user writes a software. The jenkins can 
- Compile code
- Run tests
- Build a new version of the app
- Deploy the app.

In Technical terms, Jenkins is a continuous integration(CI) and continuous deployment(CD) tool.

## What is a pipeline?
Eg Assembly of a laptop
Laptop Body -> Attach Mainboard -> Attach Keyboard -> Do tests -> Package  -> Shipment.

Now with Jenkins we dont ship physical product but softwares and softwares also need a series of steps before it is deployed to prod.

Documentation link -> https://www.jenkins.io/doc/book/pipeline/
```Groovy
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building a new Laptop...'
                sh 'mkdir -p build'
                sh 'touch build/computer.txt'
                sh 'echo "Laptop Body" >> build/computer.txt'
                sh 'cat build/computer.txt'
                sh 'echo "Attach Mainboard" >> build/computer.txt'
                sh 'cat build/computer.txt'
                sh 'echo "Attach Keyboard" >> build/computer.txt'
                sh 'cat build/computer.txt'
                sh 'echo "Do tests" >> build/computer.txt'
                sh 'cat build/computer.txt'
                sh 'echo "Package" >> build/computer.txt'
                sh 'cat build/computer.txt'
                sh 'echo "Shipping..." >> build/computer.txt'
                sh 'cat build/computer.txt'
            }
        }
    }
}

```

## Storing build artifacts in Jenkins.

```Groovy
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                cleanWs()
                echo 'Building a new Laptop...'
                sh '''
                mkdir -p build
                touch build/computer.txt
                echo "Laptop Body" >> build/computer.txt
                cat build/computer.txt
                echo "Attach Mainboard" >> build/computer.txt
                cat build/computer.txt
                echo "Attach Keyboard" >> build/computer.txt
                cat build/computer.txt
                echo "Do tests" >> build/computer.txt
                cat build/computer.txt
                echo "Package" >> build/computer.txt
                cat build/computer.txt
                echo "Shipping..." >> build/computer.txt
                cat build/computer.txt
                '''
            }
        }
    }
    
    post {
        success{
            archiveArtifacts artifacts: 'build/**'
        }
    }
}

```

## Multiple stages
Here we are using command `test -f build/computer.txt` to see if a file exists or not. 
Useful when u are trying to test using code instead of manual.
```Groovy
pipeline {
    agent any

    stages {
        stage('One') {
            steps {
                echo 'Stage 1'
                sh 'sleep 5'
            }
        }
        stage('Two'){
            steps {
                echo 'Stage 2'
                sh 'sleep 5'
                sh 'test -f build/computer.txt'
            }
        }
    }
}

```

## Defining Environment Variables

```Groovy
pipeline {
    agent any
    environment{
        BUILD_FILE_NAME = 'helloworld.js'
    }

    stages {
        stage('One') {
            steps {
                echo 'Stage 1'
                echo "$BUILD_FILE_NAME"
            }
        }
    }
}

```
