
**Simple CLI-style diagram** of Continuous Integration (CI)
```
          Developer pushes code
                  │
                  ▼
           +--------------+
           |   Git Repo   |
           +--------------+
                  │
                  ▼
        +--------------------+
        |   CI Server        |
        | (Jenkins/GitHub CI)|
        +--------------------+
          │        │        │
          ▼        ▼        ▼
   +---------+ +---------+ +---------+
   | Build   | | Test    | | Lint    |
   +---------+ +---------+ +---------+
          │        │        │
          └────┬───┴───┬────┘
               ▼       ▼
         +------------------+
         |  Integration OK? |
         +------------------+
               │
        ┌──────┴──────┐
        ▼             ▼
   +---------+   +---------+
   | Success |   | Failure |
   +---------+   +---------+
        │             │
        ▼             ▼
+----------------+  Notify Dev
| Deploy (opt.)  |
+----------------+
```

### Short explanation:

- Developer pushes code → triggers CI pipeline
- CI server runs **build, tests, and checks**
- If everything passes → optional deployment
- If something fails → developer is notified immediately

## Jenkins Build Environment

Real World Scenario -> You have an JS app which requires NPM.

```Groovy title="Checking if 'npm' is there on Jenkins Agent."
pipeline {
    agent any

    stages {
        stage('Start App') {
            steps {
                echo 'Starting App'
                sh 'npm --version'
            }
        }
    }
}
```

Problem - Jenkins doesn't have `npm` out of the box. So you need to install it.

There are 2 approaches.
1. Install all tools (like node.js/npm) on the Jenkins agent manually.
2. Use docker with containers.

```Groovy
pipeline {
    agent any

    stages {
        stage('Without Docker Installation') {
            steps {
            //These commands are run on Jenkins Agent directly
                sh 'echo "Without Docker"'
            }
        }
        stage('With Docker Installation') {
            agent{
                docker{
                    image 'node:alpine3.22'
                }
            }
            steps {
            //These commands are run inside Docker Container
                sh 'echo "With Docker"'
                sh 'npm --version'
            }
        }
    }
}

```

## Workspace Synchronization
Documentation -> https://www.jenkins.io/doc/book/pipeline/docker/#workspace-synchronization

Real World Use-Case:
- If like above case where one stage is using Docker and other doesn't then what happens is that Jenkins will create multiple workspaces which causes a problem coz these 2 stages dont have access to each other resources 
- eg common files created in Stage1 needs to be used in Stage 2 then the issue arises.

Fix
```Groovy
pipeline {
    agent any

    stages {
        stage('Without Docker Installation') {
            steps {
                sh 'echo "Without Docker"'
                sh 'touch test.txt'
            }
        }
        stage('With Docker Installation') {
            agent{
                docker{
                    image 'node:alpine3.22'
                    reuseNode true
                }
            }
            steps {
                sh 'echo "With Docker"'
                sh 'npm --version'
                sh 'ls -la'
            }
        }
    }
}

```

## Create your own pipeline using 'JenkinsFile'.

1. The file name has to be `JenkinsFile` only.
```Groovy title="JenkinsFile"
pipeline {
    agent any

    stages {
        stage('Hello') {
            steps {
                echo 'Hello World'
            }
        }
    }
}

```

2. Create a new Item/Job of type `pipeline` and then Setup the pipeline to have `Pipeline script from SCM`

#### Behind the scenes of this 
Here’s a concise summary:

- Jenkins **controller** selects an agent that supports Docker.
- The controller **clones the Git repo** into the workspace.
- The agent **starts a Docker container** and **mounts the workspace** into it.
- This mapping ensures:
    - The container can access project files
    - Any changes persist after the container stops
- The container provides a **specific environment/tools** (e.g., Node.js).
- Commands run inside the container.
- After execution
    - The container is **stopped and removed**
    - Jenkins continues the pipeline
- Logs show:
    - Repo clone → `docker run` → command execution → `docker stop`

👉 Key idea: the **workspace mapping is critical**, because containers are temporary.

## Publishing a JUnit test report

```Groovy title="JenkinsFile"
pipeline {
    agent any
    stages {
        stage('Hello') {
            steps {
                echo 'Hello World'
                sh '''
                 # ls 
                '''
            }
        }
    }
    post{
        always{
            junit 'test-results/junit.xml'
        }
    }
}

```

## Running E2E tests with Playwright (End-to-End)
```Groovy title="JenkinsFile"
pipeline {
    agent any

    stages {
        /*

        stage('Build') {
            agent {
                docker {
                    image 'node:18-alpine'
                    reuseNode true
                }
            }
            steps {
                sh '''
                    ls -la
                    node --version
                    npm --version
                    npm ci
                    npm run build
                    ls -la
                '''
            }
        }
        */

        stage('Test') {
            agent {
                docker {
                    image 'node:18-alpine'
                    reuseNode true
                }
            }

            steps {
                sh '''
                    #test -f build/index.html
                    npm test
                '''
            }
        }

        stage('E2E') {
            agent {
                docker {
                    image 'mcr.microsoft.com/playwright:v1.39.0-jammy'
                    reuseNode true
                }
            }

            steps {
                sh '''
                    npm install serve
                    node_modules/.bin/serve -s build &
                    sleep 10
                    npx playwright test
                '''
            }
        }
    }

    post {
        always {
            junit 'jest-results/junit.xml'
        }
    }
}
```

## Running stages in parallel

```Groovy title="JenkinsFile"
pipeline {
    agent any

    stages {
        /*

        stage('Build') {
            agent {
                docker {
                    image 'node:18-alpine'
                    reuseNode true
                }
            }
            steps {
                sh '''
                    ls -la
                    node --version
                    npm --version
                    npm ci
                    npm run build
                    ls -la
                '''
            }
        }
        */

        stage('Tests') {
            parallel {
                stage('Unit tests') {
                    agent {
                        docker {
                            image 'node:18-alpine'
                            reuseNode true
                        }
                    }

                    steps {
                        sh '''
                            #test -f build/index.html
                            npm test
                        '''
                    }
                    post {
                        always {
                            junit 'jest-results/junit.xml'
                        }
                    }
                }

                stage('E2E') {
                    agent {
                        docker {
                            image 'mcr.microsoft.com/playwright:v1.39.0-jammy'
                            reuseNode true
                        }
                    }

                    steps {
                        sh '''
                            npm install serve
                            node_modules/.bin/serve -s build &
                            sleep 10
                            npx playwright test  --reporter=html
                        '''
                    }

                    post {
                        always {
                            publishHTML([allowMissing: false, alwaysLinkToLastBuild: false, keepAll: false, reportDir: 'playwright-report', reportFiles: 'index.html', reportName: 'Playwright HTML Report', reportTitles: '', useWrapperFileDirectly: true])
                        }
                    }
                }
            }
        }
    }
}
```
