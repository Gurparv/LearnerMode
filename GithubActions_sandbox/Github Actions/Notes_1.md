Github Action -> is a continous integration and continous delivery platform that allows you to automate your build
test and deployement pipeline.
And Github Action Workflow is a process that you set up in your repo to automate software developement lifecycle tasks.
Github Actions -> goes beyond just DevOps and lets you run workflows when other events happen in your repository.

# Github Actions Workflow :
 - A Workflow is a configurable automated process that will run one or more jobs.
 - Workflows are defined by a YAML file checked into your repo and will run when triggered by an event in your repo, or they can be triggered manually , or at a defined schedule.
 - WIth a workflow, you can build , test , package , release and deploy any project on Github.
