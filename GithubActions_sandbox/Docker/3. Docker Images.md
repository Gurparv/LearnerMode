## Whats an image
- App binaries and dependencies
- Meta about the image data and how to run the image 
	- Command to see it -> `docker image inspect <image_tag_or_imageid>` 
- Not a complete OS. No Kernel, kernel modules(eg drivers)
- Small as one file(your app binary) like a golang static binary
- Big as a Ubuntu distro with apt, and Apache, PHP, and more installed.

---
## Image Cache

Images are designed using the **union files system** of about making layers about the changes.

```Bash
# Shows layers of changes made in image.
docker image history <image_tag_or_imageid>

## returns JSON metadata about the image
docker image inspect <image_tag_or_imageid>
```

## Building Images
#### 1. The Dockerfiles Basics 
Dockerfile contains the instructions on how to build our image

Every dockerfile has below commands
```
- FROM
- ENV
- RUN
- EXPOSE
- CMD
- WORKDIR
- COPY
```

#### 2. Running Docker Builds
Below command will execute line by line content of your Dockerfile inside the Docker engine and cache each of those layers. 
```Bash
# builds the Dockerfile in current directory
docker image build --tag <your_custom_tag> .
```
