### Image vs Container
**Image** -> The application we want to run. (the whole files, binaries , libraries included)
**Container** -> A Container is an instance of the image running as a process.
**Registry** -> Docker's default image "registry" is called Docker Hub (hub.docker.com).

---
### Starting a Nginx Web Server
```Bash
docker container run --publish 8080:80 nginx

# OR 
docker container run --publish 8080:80 --detach nginx
```

#### Whats happening when we use this command.
1. Downloaded image 'nginx' from Docker Hub.
2. Started a new container from that image
3. Opened port 8080 on the host IP
4. Routes that traffic to the container IP, port 80

## Run vs Start
```bash
# Always starts a new container
docker container run 

# This starts an existing stopped one.
docker container start <container_id> 
```

## Other commands
```Bash
# Lists running processes inside a running container
docker top <container_name_or_id>

# Check logs of a container
docker container logs <container_name_or_id>
```

---

## What happens in 'docker container run'
1. Looks for that image locally in image cache, doesnt find anything.
2. Then looks in remote image repository (defaults to Docker Hub)
3. Downloads the latest version(nginx:latest by default)
4. Creates a new container based on that image and prepares to start.
5. Gives it a virtual IP on a private network inside docker enginer.
6. Opens up specified port(8080 in our case) on host and forwards to specified port(80) in container
7. Starts container by using the CMD in the image Dockerfile.

```Bash
docker container run --publish 8080:80 --name myWebApp -d nginx:1.11 nginx -T
```

## Getting a Shell Inside Containers (*Important*)

```Bash
# starts a new container interactively.
docker container run -it 

# run additional command in existing container
# -t it pseudo-tty (simulates a real terminal,like waht SSH does)
# -i is short for 'interactive'
docker container exec -it

# Useful if you use Different Linux distros in containers. Please note this is not the same as SSH
```

Example: this is very minimal version of Ubuntu image.
```Bash
docker container run -it ubuntu 

# You can stuff to it using bash
apt-get install -y curl

docker container start -ai ubuntu
```

In this you can download packages from the internet etc .. this is your ubuntu container.
