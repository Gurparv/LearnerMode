

## ❌ Problem

UI tests are:

- slow
- flaky
- dependent on UI
---

## ✅ Shift

> UI = validation  
> API = setup

---

## 🔧 Example

❌ Old way:

```ts
await page.click('Create User');
await page.fill(...);
```

---

✅ New way:

```ts
await api.createUser();
await page.goto('/users');
await expect(userRow).toBeVisible();
```

---

## 🔥 Benefits

|Metric|Before|After|
|---|---|---|
|Speed|slow|fast|
|Stability|flaky|stable|
|Debugging|hard|easy|

---

## 🔥 Hybrid Model

|Layer|Responsibility|
|---|---|
|API|setup|
|UI|validation|
|DB|verification (optional)|

---

## 🔥 Real Example

**Checkout Flow**

❌ UI-only:
- add to cart
- fill address
- payment

✅ API-first:

```ts
await api.createCart();
await api.addItem();
await page.goto('/checkout');
```
