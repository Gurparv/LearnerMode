

## ❌ Problem

- Tests collide
- Parallel runs break
- Data pollution

---

## ✅ Principle

> Every test owns its data

---

## 🔧 Factory Pattern

```ts
class UserFactory {
  static create() {
    return {
      email: `user_${Date.now()}@test.com`,
      password: 'Password123'
    };
  }
}
```

---

## 🔧 API-Based Setup (FAST + STABLE)

```ts
const user = UserFactory.create();
await api.createUser(user);
```

---

## 🔧 Cleanup Strategy

Option 1 (best):

- Auto-delete via API

Option 2:

- TTL data (expires automatically)

---

## 🔥 Isolation Levels

|Level|Strategy|
|---|---|
|Basic|unique data|
|Better|per-test user|
|Elite|per-test environment|

---

## 🔥 Real Example

❌ Bad:

```ts
login("testuser")
```

✅ Good:

```ts
const user = await createUser();
login(user);
```
