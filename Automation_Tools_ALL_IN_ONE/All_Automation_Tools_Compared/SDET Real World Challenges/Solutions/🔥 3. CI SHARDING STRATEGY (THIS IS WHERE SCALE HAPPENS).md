

## ❌ Problem

- 1000 tests → 1 hour runtime
- uneven load
    

---

## ✅ Goal

> Finish in minutes, not hours

---

## 🔧 Naive Sharding (BAD)

```yaml
shard: [1,2,3,4]
```

👉 equal split → uneven runtime

---

## 🔥 Smart Sharding (ELITE)

### Step 1: Collect test durations

```ts
testInfo.duration
```

Store historical timing

---

### Step 2: Balance shards

Example:

|Test|Time|
|---|---|
|A|10s|
|B|2s|
|C|8s|

👉 distribute evenly

---

## 🔧 GitHub Actions Example

```yaml
strategy:
  matrix:
    shardIndex: [1,2,3]
    shardTotal: [3]

steps:
  - run: npx playwright test --shard=${{matrix.shardIndex}}/${{matrix.shardTotal}}
```

---

## 🔥 Advanced

- run slow tests separately
- prioritize critical tests first

---

## 🧠 Insight

> You are optimizing **time distribution**, not just splitting tests
