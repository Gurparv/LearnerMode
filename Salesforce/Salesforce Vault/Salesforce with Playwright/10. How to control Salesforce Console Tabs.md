10. How to control Salesforce Console Tabs using the Workspace API in Playwright


In **Salesforce Lightning Console apps**, records open in **workspace tabs and subtabs**. If you automate them like normal pages, tests become flaky because Playwright still sees **one browser page**, while Salesforce internally manages tabs.

The correct way is to interact with the **Salesforce Workspace API** and trigger it through `page.evaluate()` in Playwright.

Below is the **practical guide used in real Salesforce automation frameworks**.

---

# 1. Wait Until Workspace API Is Available

The Workspace API loads only after the console app finishes rendering.

```javascript
await page.waitForSelector('lightning-workspace-api', { timeout: 60000 });
```

This ensures the console environment is ready.

---

# 2. Get All Open Console Tabs

This is useful when previous sessions restore tabs.

```javascript
const tabs = await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  return await workspaceAPI.getAllTabInfo();
});

console.log(tabs);
```

Typical response:

```json
[
 { "tabId": "workspace-tab-1", "title": "Accounts" },
 { "tabId": "workspace-tab-2", "title": "Test Account" }
]
```

You can use these IDs to control tabs.

---

# 3. Close All Existing Tabs (Clean Session)

This is extremely useful before starting each test.

```javascript
await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  const tabs = await workspaceAPI.getAllTabInfo();

  for (const tab of tabs) {
    if (!tab.isPinned) {
      await workspaceAPI.closeTab({ tabId: tab.tabId });
    }
  }
});
```

This ensures your test **always starts with a clean console workspace**.

---

# 4. Get the Currently Active Tab

Sometimes Salesforce opens subtabs automatically.

```javascript
const activeTab = await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  return await workspaceAPI.getFocusedTabInfo();
});

console.log(activeTab.tabId);
```

---

# 5. Switch to Another Tab

If multiple records are open.

```javascript
await page.evaluate(async (tabId) => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  await workspaceAPI.focusTab({ tabId });
}, tabId);
```

Example:

```javascript
await focusTab('workspace-tab-2');
```

---

# 6. Open a Record in a New Console Tab

Instead of clicking through UI navigation.

```javascript
await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');

  await workspaceAPI.openTab({
    url: '/lightning/r/Account/001XXXXXXXXXXXX/view',
    focus: true
  });
});
```

This is **much faster and more reliable**.

---

# 7. Open a Subtab Under Parent Tab

Many Salesforce records open as **subtabs under objects**.

```javascript
await page.evaluate(async (parentTabId) => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');

  await workspaceAPI.openSubtab({
    parentTabId,
    url: '/lightning/r/Contact/003XXXXXXXXXXXX/view',
    focus: true
  });
}, parentTabId);
```

---

# 8. Highlight a Tab (Helpful for Debugging)

This visually marks the tab.

```javascript
await page.evaluate(async (tabId) => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');

  await workspaceAPI.setTabHighlighted({
    tabId,
    highlighted: true
  });
}, tabId);
```

---

# 9. Get Parent Tab ID

When working with subtabs.

```javascript
const parentTab = await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  const tab = await workspaceAPI.getFocusedTabInfo();
  return tab.parentTabId;
});
```

---

# 10. Real Test Example (Clean + Open Record)

```javascript
test('Open Account in console tab', async ({ page }) => {

  await page.goto('/lightning/page/home');

  await page.waitForSelector('lightning-workspace-api');

  // clean session
  await page.evaluate(async () => {
    const workspaceAPI = document.querySelector('lightning-workspace-api');
    const tabs = await workspaceAPI.getAllTabInfo();

    for (const tab of tabs) {
      if (!tab.isPinned) {
        await workspaceAPI.closeTab({ tabId: tab.tabId });
      }
    }
  });

  // open record
  await page.evaluate(async () => {
    const workspaceAPI = document.querySelector('lightning-workspace-api');

    await workspaceAPI.openTab({
      url: '/lightning/r/Account/001XXXXXXXXXXXX/view',
      focus: true
    });
  });

});
```

# Best Framework Pattern

Most Salesforce automation frameworks structure it like this:

```
test.beforeEach
   ↓
closeAllConsoleTabs()
   ↓
openRequiredRecordTab()
   ↓
runTest()
```

This eliminates **90% of console-related flakiness**.

---

# Pro Tip (Very Important)

Sometimes `lightning-workspace-api` isn't accessible through `document.querySelector`.

Then use the **Aura event bridge**:

```javascript
window.postMessage({
  category: 'workspaceAPI',
  method: 'getAllTabInfo'
});
```

But most orgs work fine with the direct selector.
