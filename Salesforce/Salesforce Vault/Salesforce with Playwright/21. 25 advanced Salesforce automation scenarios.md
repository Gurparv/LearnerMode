21. 25 advanced Salesforce automation scenarios that break most test frameworks (and how to handle them in Playwright)

Below are **25 advanced Salesforce automation scenarios** that frequently **break UI automation frameworks** (especially in Lightning + Console apps) and the **Playwright strategies used by senior Salesforce QA engineers to handle them**.

These are real-world issues seen in **large enterprise orgs**.

---

# 1. Lightning DOM Re-Rendering After Save

**Problem:**  
Aura/LWC components re-render after actions → stale element errors.

**Solution**

Use locators + wait for Lightning idle.

```javascript
await page.getByRole('button', { name: 'Save' }).click();

await page.locator('.slds-spinner').waitFor({ state: 'hidden' });
```

---

# 2. Salesforce Console Tabs Instead of Browser Tabs

Console apps use **workspace tabs**, not browser tabs.

**Solution:** Workspace API.

```javascript
await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  const tabs = await workspaceAPI.getAllTabInfo();
});
```

---

# 3. Multiple Console Tabs Open From Previous Sessions

Salesforce restores tabs automatically.

**Solution:** Close tabs before each test.

```javascript
await workspaceService.closeAllTabs();
```

---

# 4. Salesforce Dynamic IDs

Lightning generates IDs dynamically.

Bad selector:

```
#input-392
```

Good selector:

```javascript
page.getByLabel('Account Name')
```

---

# 5. Slow List Views with Large Data

List views may contain **thousands of records**.

**Solution**

Avoid UI navigation.

Use direct URL:

```
/lightning/r/Account/{recordId}/view
```

---

# 6. Salesforce Flow Screen Transitions

Flows re-render entire components.

**Solution**

Wait for flow container.

```javascript
await page.locator('lightning-flow').waitFor();
```

---

# 7. Toast Notifications Blocking Clicks

Lightning shows toast messages.

**Solution**

```javascript
await page.locator('.toastMessage').waitFor();
await page.locator('.toastMessage').waitFor({ state: 'hidden' });
```

---

# 8. Approval Processes With Multiple Users

Approval requires different users.

**Solution**

Multiple Playwright sessions.

```
auth/
 salesUser.json
 managerUser.json
```

Test:

```javascript
test.use({ storageState: 'auth/managerUser.json' });
```

---

# 9. Salesforce Async Flows and Triggers

Some automation runs asynchronously.

**Solution**

Use polling via API.

```javascript
await waitForRecord(
  request,
  `SELECT Status FROM Quote WHERE Id='${quoteId}'`
);
```

---

# 10. Record Created But UI Not Updated

UI sometimes delays updates.

**Solution**

Verify via API.

```javascript
SELECT Name FROM Account WHERE Id='recordId'
```

---

# 11. Dynamic Related Lists

Related lists reload asynchronously.

**Solution**

Scope locator.

```javascript
await page
  .locator('records-related-list')
  .getByRole('link', { name: 'Test Opportunity' })
  .click();
```

---

# 12. File Upload in Salesforce

Salesforce file uploader uses hidden inputs.

**Solution**

```javascript
await page.setInputFiles(
  'input[type="file"]',
  'test.pdf'
);
```

---

# 13. Lightning Shadow DOM Components

Many LWC components use shadow DOM.

Playwright handles this automatically via locators.

```javascript
await page
  .locator('records-lwc-highlights-panel')
  .getByRole('button', { name: 'Edit' })
  .click();
```

---

# 14. Record Locking During Approval

Records become locked.

**Solution**

Verify lock via API.

```sql
SELECT IsLocked FROM Record WHERE Id='recordId'
```

---

# 15. Lightning Page Layout Changes

Admins change layouts frequently.

**Solution**

Metadata-driven validation.

Query metadata:

```
FieldDefinition
Layout
```

Generate tests automatically.

---

# 16. Quote → DocuSign Integration

External signing required.

**Solution**

Validate envelope via API.

```sql
SELECT Status FROM DocuSignEnvelope__c WHERE Quote__c='quoteId'
```

---

# 17. Order Creation After Quote Signing

After DocuSign completion.

Verify via API.

```sql
SELECT Id FROM Order WHERE QuoteId='quoteId'
```

---

# 18. Salesforce Search Is Slow

Global search takes time.

**Solution**

Avoid search UI.

Navigate directly.

```
/lightning/r/ObjectName/recordId/view
```

---

# 19. Salesforce Permissions / Profiles

Different users see different UI.

**Solution**

Data-driven role testing.

Example matrix:

```
Sales User
Manager
Support
Admin
```

---

# 20. Bulk Data Operations

Large orgs may contain millions of records.

**Solution**

Use filtered SOQL.

```sql
SELECT Id FROM Account WHERE Name='Test Account'
```

---

# 21. Salesforce Sandbox Refresh Breaks Tests

Sandbox refresh resets data.

**Solution**

Use **API test data factory**.

```javascript
const scenario = await factory.createOpportunityScenario();
```

---

# 22. Email Notifications From Workflows

Emails triggered after events.

**Solution**

Validate via API or email service logs.

Example:

```
EmailMessage object
```

---

# 23. Scheduled Jobs / Batch Apex

Jobs run asynchronously.

**Solution**

Query async job status.

```sql
SELECT Status FROM AsyncApexJob
```

---

# 24. Lightning Page Load Delays

Lightning apps load slowly.

**Solution**

Wait for navigation container.

```javascript
await page.waitForSelector('one-appnav');
```

---

# 25. Salesforce Deployment Changes UI

New deployments modify UI.

**Solution**

Combine:

```
Metadata testing
Flow testing
UI automation
```

Framework automatically detects:

```
New fields
Removed layouts
Changed flows
```
