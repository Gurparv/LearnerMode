12. How to automatically test all Profiles in your Salesforce org without manually creating tests for each profile.

In large Salesforce orgs you might have **30–200 Profiles**, and writing separate UI tests for each is not scalable. The advanced strategy is to **discover profiles dynamically from Salesforce via API, map them to test users, and run the same Playwright test logic across them automatically**.

This is commonly called **Dynamic Role/Profile Testing**.

---

# 1. Fetch All Profiles Using Salesforce API

Salesforce allows querying metadata with **SOQL via REST API**.

Example query:

```sql
SELECT Id, Name FROM Profile WHERE UserType='Standard'
```

In Playwright (using APIRequestContext):

```javascript
async function getProfiles(request) {
  const response = await request.get(
    `/services/data/v60.0/query?q=SELECT+Id,Name+FROM+Profile+WHERE+UserType='Standard'`
  );

  const data = await response.json();
  return data.records;
}
```

Result example:

```json
[
 { "Id": "00e1...", "Name": "Sales Profile" },
 { "Id": "00e2...", "Name": "Support Profile" },
 { "Id": "00e3...", "Name": "Marketing Profile" }
]
```

Now your test suite **knows every profile automatically**.

---

# 2. Map Profiles to Test Users

Each profile must have a **test user**.

Option A (recommended): **User factory**

Create users dynamically:

```javascript
async function createUser(request, profileId) {

  const username = `testuser_${Date.now()}@example.com`;

  const response = await request.post(
    '/services/data/v60.0/sobjects/User',
    {
      data: {
        Username: username,
        Alias: 'tuser',
        Email: username,
        ProfileId: profileId,
        TimeZoneSidKey: 'America/Los_Angeles',
        LocaleSidKey: 'en_US',
        EmailEncodingKey: 'UTF-8',
        LanguageLocaleKey: 'en_US',
        LastName: 'Automation'
      }
    }
  );

  return response.json();
}
```

Option B: Maintain a **profile-user mapping file**.

Example:

```javascript
export const profileUsers = {
  "Sales Profile": "sales.user@test.com",
  "Support Profile": "support.user@test.com",
  "System Administrator": "admin@test.com"
};
```

---

# 3. Generate Playwright Tests Dynamically

Instead of writing multiple tests, generate them programmatically.

Example:

```javascript
const profiles = await getProfiles(request);

for (const profile of profiles) {

  test.describe(`Flow test for ${profile.Name}`, () => {

    test(`Validate Opportunity flow`, async ({ page }) => {

      await loginAsProfile(profile.Name, page);

      await page.goto('/lightning/o/Opportunity/list');

      await page.getByRole('button', { name: 'New' }).click();

      await expect(page.locator('lightning-flow')).toBeVisible();

    });

  });

}
```

Now **one test automatically runs for every profile**.

---

# 4. Filter Profiles (Important)

Not all profiles should run UI tests.

Common exclusions:

- Guest User
    
- Integration User
    
- Community Profiles
    

Filter example:

```javascript
const filteredProfiles = profiles.filter(p =>
  !p.Name.includes('Guest') &&
  !p.Name.includes('Integration')
);
```

# 5. Run Each Profile Test in Parallel ⚡

Playwright config:

```javascript
workers: 6
```

Example execution:

```
Sales Profile → worker 1
Support Profile → worker 2
Marketing Profile → worker 3
Admin → worker 4
```

Even with **30 profiles**, tests finish quickly.

# 6. Validate Profile-Based Behavior

Profiles affect:

- Object access
    
- Field visibility
    
- Flow steps
    
- Buttons
    

Example validations:

### Flow visible

```javascript
await expect(page.locator('lightning-flow')).toBeVisible();
```

### Button hidden

```javascript
await expect(page.getByRole('button', { name: 'Delete' })).not.toBeVisible();
```

### Field read-only

```javascript
await expect(page.getByLabel('Discount')).toBeDisabled();
```

# 7. Capture Permission Failures Automatically

Instead of failing tests, you can log access problems.

Example:

```javascript
if (await page.getByText('Insufficient Privileges').isVisible()) {
  console.log(`Profile ${profile.Name} cannot access flow`);
}
```

This becomes a **security audit test**.
