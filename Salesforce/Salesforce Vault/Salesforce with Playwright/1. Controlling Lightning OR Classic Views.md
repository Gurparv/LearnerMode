
- Classic 
	- URL -> `https://cunning-panda-eu9k4v-dev-ed.trailblaze.my.salesforce.com/home/home.jsp?source=lex`
- Lightning 
	- URL -> `https://cunning-panda-eu9k4v-dev-ed.trailblaze.lightning.force.com/lightning/page/home`

==Lesson 1

As an automation engineer, you must **control the URL + enforce UI mode**, not rely on defaults.

### 1. Force Lightning Experience.

Use the **Lightning domain URL** directly:
```
https://yourDomain.lightning.force.com/lightning/page/home

Eg
https://cunning-panda-eu9k4v-dev-ed.trailblaze.lightning.force.com/lightning/page/home
```

For Specific Pages:

1. Accounts:
```Code
/lightning/o/Account/list
```

2. For Specific Record
```Code
/lightning/r/Account/<recordId>/view
```

### Best Practice 
After login, explicitly navigate:
```Typescript

await page.goto('https://yourDomain.lightning.force.com/lightning/page/home');
```
This forces Lightning every time.

### 2. Force Salesforce Classic:

Use the Classic domain:
```Code
https://yourDomain.my.salesforce.com/home/home.jsp

Eg
https://cunning-panda-eu9k4v-dev-ed.trailblaze.my.salesforce.com/home/home.jsp
```

Or:
```Code
https://yourDomain.my.salesforce.com/001/o   // Accounts tab
```

if user lands in Lightning, force Classic:
```Code
https://yourDomain.my.salesforce.com/ltng/switcher?destination=classic
```

Then navigate to your target page.

### 3.  Prevent Mode Switching Issues (Important)
Option A -- Dedicated Users (Recommended)
Create:
- One user with Lightning enabled
- One user set to Classic only

Control it via Profile Settings

Option B -- Detect + Enforce
In playwright
```Typescript

if (page.url().includes('lightning')) {
  // already Lightning
} else {
  await page.goto('https://yourDomain.lightning.force.com/lightning/page/home');
}
```

### Automation Best Practice (Most Stable)
Always:
1. Login
2. Wait for successful redirect
3. Explicity `page.goto()` the exact URL you want
4. Assert URL contains `lightning` or `home.jsp`

Never depend on:
- Last used UI
- Default org setting
- User preference

==Control the URL → You control the UI.
