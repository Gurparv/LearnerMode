18. How to automatically generate Playwright tests from Salesforce metadata (Objects, Fields, and Flows) so your framework detects UI changes automatically.

In large Salesforce orgs (hundreds of objects, fields, flows), UI automation often breaks because **admins add/remove fields, change layouts, or update flows**.

A powerful approach is **Metadata-Driven Testing**: your framework **reads Salesforce metadata through the API and generates Playwright tests dynamically**.  
This allows your automation to **detect UI changes automatically**.

Instead of hardcoding tests, the framework **discovers objects, fields, and flows at runtime**.

# 2. Fetch Salesforce Metadata via API

Use the **Tooling API or REST API**.

Example: get all objects.

```javascript
async function getObjects(request) {
  const response = await request.get(
    `/services/data/v60.0/sobjects`
  );

  const data = await response.json();
  return data.sobjects;
}
```

Example result:

```
Account
Opportunity
Case
Lead
CustomObject__c
```

---

# 3. Fetch Fields for Each Object

Example SOQL query:

```sql
SELECT QualifiedApiName, DataType 
FROM FieldDefinition 
WHERE EntityDefinition.QualifiedApiName = 'Account'
```

Implementation:

```javascript
async function getFields(request, objectName) {
  const response = await request.get(
    `/services/data/v60.0/tooling/query?q=
    SELECT+QualifiedApiName,DataType+
    FROM+FieldDefinition+
    WHERE+EntityDefinition.QualifiedApiName='${objectName}'`
  );

  const data = await response.json();
  return data.records;
}
```

This gives you **every field in the object**.

---

# 4. Generate Playwright Tests Dynamically

Instead of writing tests manually, create them from metadata.

Example generator:

```javascript
const objects = await getObjects(request);

for (const object of objects) {

  test(`Open ${object.name} list view`, async ({ page }) => {

    await page.goto(`/lightning/o/${object.name}/list`);

    await expect(page.getByRole('heading')).toContainText(object.label);

  });

}
```

Now your suite automatically tests **every object page**.

---

# 5. Automatically Validate Fields on Record Pages

Generate tests to ensure fields appear in UI.

Example:

```javascript
const fields = await getFields(request, 'Account');

for (const field of fields) {

  test(`Account field visible: ${field.QualifiedApiName}`, async ({ page }) => {

    await page.goto('/lightning/o/Account/new');

    await expect(page.getByLabel(field.QualifiedApiName))
      .toBeVisible();

  });

}
```

If an admin removes a field from the layout, the test fails → **UI change detected**.

# 6. Generate Flow Tests Automatically

Salesforce stores flows as metadata.

Query flows:

```sql
SELECT Id, DeveloperName 
FROM FlowDefinitionView
```

Implementation:

```javascript
async function getFlows(request) {
  const response = await request.get(
    `/services/data/v60.0/tooling/query?q=
    SELECT+DeveloperName+FROM+FlowDefinitionView`
  );

  return (await response.json()).records;
}
```

Generate tests:

```javascript
for (const flow of flows) {

  test(`Flow launches: ${flow.DeveloperName}`, async ({ page }) => {

    await page.goto(`/lightning/flow/${flow.DeveloperName}`);

    await expect(page.locator('lightning-flow')).toBeVisible();

  });

}
```

Now every flow has a **basic health check**.

# 7. Detect Layout Changes Automatically

Admins often modify **page layouts**.

You can query layouts:

```sql
SELECT Layout.Name, EntityDefinition.QualifiedApiName 
FROM Layout
```

Use this metadata to confirm expected fields exist.

Example test:

```
Account Layout
  Name
  Phone
  Industry
```

Generated test verifies those fields appear in UI.

# 8. Store Metadata Snapshots

To detect changes over time, store metadata snapshots.

Example:

```

  fields.json
  flows.json
```

Each CI run compares metadata.

If differences appear:

```

New object added
Flow changed
```

The framework generates new tests automatically.

```javascript
  }

```

One file → **hundreds of tests generated automatically**.
