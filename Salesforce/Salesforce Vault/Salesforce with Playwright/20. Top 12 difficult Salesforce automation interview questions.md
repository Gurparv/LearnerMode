# 1. How do you handle Salesforce Lightning dynamic DOM in UI automation?

**Problem:**  
Lightning (Aura/LWC) constantly re-renders components, causing flaky tests.

**Best practice**

- Use **Playwright locators (auto-retry)**
    
- Avoid static element handles
    
- Wait for Lightning stability signals
    

Example:

```javascript
await page.locator('records-lwc-highlights-panel')
  .getByRole('button', { name: 'Edit' })
  .click();
```

Use helper:

```javascript
await waitForLightningIdle(page);
```

# 2. How do you test Salesforce console apps with multiple tabs?

Console apps open **workspace tabs + subtabs**, not browser tabs.

Solution: **Workspace API**

Example:

```javascript
await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');

  const tabs = await workspaceAPI.getAllTabInfo();

  for (const tab of tabs) {
    if (!tab.isPinned) {
      await workspaceAPI.closeTab({ tabId: tab.tabId });
    }
  }
});
```

This ensures **clean session state** before each test.

# 4. How do you verify Salesforce data without using the UI?

Use **SOQL queries via REST API**.

Example:

```javascript
const response = await request.get(
  `/services/data/v60.0/query?q=
  SELECT+Name+FROM+Account+WHERE+Id='${accountId}'`
);

const data = await response.json();

expect(data.records[0].Name).toBe("Test Account");
```

This avoids:

```
List view search
Opening records
Scrolling UI
```

# 7. How do you automate role-based behavior in Salesforce?

Salesforce is heavily **profile / permission based**.

Strategy:

- Multiple user sessions
    
- Data-driven tests
    
- Separate `storageState` files
    

Example:

```
auth/
  salesUser.json
  managerUser.json
  supportUser.json
```

Test:

```javascript
test.use({ storageState: 'auth/managerUser.json' });
```

# 12. How do you prevent flaky Salesforce tests?

Most flakiness comes from:

```
Lightning re-renders
Async flows
Console tab changes
Slow page loads
```

Solutions:

- wait for Lightning idle
    
- avoid fixed waits
    
- verify DOM stability
    

Example:

```javascript
await page.locator('.slds-spinner')
  .waitFor({ state: 'hidden' });
```

Or:

```javascript
await expect(
  page.locator('records-lwc-highlights-panel')
).toContainText('Account');
```
