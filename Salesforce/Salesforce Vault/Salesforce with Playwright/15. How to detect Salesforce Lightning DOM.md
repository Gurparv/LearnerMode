15. How to detect Salesforce Lightning DOM re-renders automatically in Playwright so your tests never click stale elements.

Salesforce **Lightning (Aura + LWC)** frequently **re-renders the DOM after actions** like saving a record, switching console tabs, or updating fields. During this re-render:

- elements temporarily disappear
    
- DOM nodes are replaced
    
- Playwright references become stale
    

This causes errors like:

```text
Element is not attached to the DOM
Timeout waiting for element
Click intercepted
```

The solution is to **detect Lightning re-renders and wait until the UI stabilizes before interacting again**.

Below are the techniques used in **robust Salesforce Playwright frameworks**.

---

# 1. Always Use Playwright Locators (Not Stored Elements)

Playwright locators automatically re-query the DOM.

❌ Bad (stale element risk)

```javascript
const saveButton = await page.$('button[title="Save"]');
await saveButton.click();
```

✅ Good

```javascript
await page.getByRole('button', { name: 'Save' }).click();
```

Locators automatically retry after DOM updates.

---

# 2. Wait for Lightning Spinners to Disappear

Lightning shows **SLDS spinners** during rerenders.

Selector:

```text
.slds-spinner
```

Implementation:

```javascript
async function waitForLightningIdle(page) {
  await page.locator('.slds-spinner').waitFor({ state: 'hidden' });
}
```

Usage:

```javascript
await page.getByRole('button', { name: 'Save' }).click();
await waitForLightningIdle(page);
```

---

# 3. Wait for Key Lightning Components to Stabilize

Important components signal that rendering finished.

Examples:

```text
records-lwc-highlights-panel
force-record-layout
lightning-record-form
```

Example wait:

```javascript
await page.waitForSelector('records-lwc-highlights-panel');
```

This ensures the **record view fully loaded**.

---

# 4. Detect DOM Re-render Using MutationObserver

For advanced stability you can detect DOM changes automatically.

Inject a DOM observer:

```javascript
async function waitForDomStable(page) {
  await page.evaluate(() => {
    return new Promise(resolve => {

      let timeout;

      const observer = new MutationObserver(() => {
        clearTimeout(timeout);

        timeout = setTimeout(() => {
          observer.disconnect();
          resolve();
        }, 500);
      });

      observer.observe(document.body, {
        childList: true,
        subtree: true
      });

      setTimeout(() => {
        observer.disconnect();
        resolve();
      }, 2000);

    });
  });
}
```

Usage:

```javascript
await page.getByRole('button', { name: 'Save' }).click();
await waitForDomStable(page);
```

This waits until **DOM changes stop**.

---

# 5. Wait for Lightning Toast Messages

Many actions trigger a toast notification.

Example:

```javascript
await page.locator('.toastMessage').waitFor();
```

Then:

```javascript
await page.locator('.toastMessage').waitFor({ state: 'hidden' });
```

This signals that the **transaction completed**.

---

# 6. Wait for URL or Record ID Change

Lightning updates the route when navigation occurs.

Example:

```javascript
await page.waitForURL(/\/lightning\/r\/Account\/.*\/view/);
```

This confirms the new record page loaded.

---

# 7. Build a Salesforce Stability Helper

Enterprise frameworks create a reusable utility.

Example:

```javascript
export async function waitForSalesforceStable(page) {

  await page.waitForLoadState('domcontentloaded');

  await page.locator('.slds-spinner')
    .waitFor({ state: 'hidden' })
    .catch(() => {});

  await page.waitForTimeout(300);

}
```

Usage everywhere:

```javascript
await page.getByRole('button', { name: 'Save' }).click();

await waitForSalesforceStable(page);
```

---

# 8. Scope Locators to Prevent Stale Clicks

Lightning re-renders sections independently.

Instead of:

```javascript
await page.getByRole('button', { name: 'Edit' }).click();
```

Scope it:

```javascript
await page
  .locator('records-lwc-highlights-panel')
  .getByRole('button', { name: 'Edit' })
  .click();
```

Scoped locators survive partial rerenders.

---

# 9. Avoid `waitForTimeout`

Many Salesforce test suites use:

```javascript
await page.waitForTimeout(5000);
```

This is unreliable because rerender time varies.

Better:

```javascript
await expect(page.getByRole('button', { name: 'Save' })).toBeEnabled();
```

---

# 10. Best Practice Pattern

Stable interaction flow:

```javascript
await page.getByRole('button', { name: 'Save' }).click();

await waitForLightningIdle(page);

await waitForDomStable(page);

await expect(
  page.locator('records-lwc-highlights-panel')
).toContainText('Updated Account');
```

This prevents almost all **stale element errors**.
