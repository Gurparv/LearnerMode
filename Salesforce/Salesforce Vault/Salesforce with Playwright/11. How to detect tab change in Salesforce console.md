11. How to detect tab change in Salesforce console and wait for the correct record to load.

In a **Salesforce Lightning Console app**, switching tabs does **not reload the browser page**. The URL may change, but the DOM is often **re-rendered asynchronously** by Aura/LWC.

So Playwright does **not automatically know** when a tab switch has completed.  
To make tests stable, you must detect the **tab focus change + record load signal**.

Below are the **reliable strategies used in Salesforce automation frameworks**.

---

# 1. Detect Tab Change Using Workspace API (Most Reliable) ⭐

Use the Workspace API to get the **focused tab info**.

```javascript
async function getFocusedTab(page) {
  return await page.evaluate(async () => {
    const workspaceAPI = document.querySelector('lightning-workspace-api');
    return await workspaceAPI.getFocusedTabInfo();
  });
}
```

Example usage:

```javascript
const tab = await getFocusedTab(page);
console.log(tab.title);
console.log(tab.recordId);
```

This tells you **which record/tab Salesforce actually focused**.

---

# 2. Wait Until Focused Tab Matches Expected Record

When opening or switching tabs:

```javascript
async function waitForRecordTab(page, recordId) {
  await page.waitForFunction(
    async (recordId) => {
      const workspaceAPI = document.querySelector('lightning-workspace-api');
      const tab = await workspaceAPI.getFocusedTabInfo();
      return tab.recordId === recordId;
    },
    recordId,
    { timeout: 20000 }
  );
}
```

Usage:

```javascript
await waitForRecordTab(page, '001XXXXXXXXXXXX');
```

This ensures **Salesforce finished switching tabs**.

---

# 3. Wait for Record Page Components to Render

Even after the tab switch, Lightning components load gradually.

A very stable signal is the **record highlights panel**.

```javascript
await page.waitForSelector('records-lwc-highlights-panel');
```

Or verify the record title:

```javascript
await expect(page.locator('records-lwc-highlights-panel'))
  .toContainText('Test Account');
```

---

# 4. Detect URL Change (Secondary Signal)

Lightning updates the URL when a record loads.

Example:

```javascript
await page.waitForURL(/\/lightning\/r\/Account\/.*\/view/);
```

This confirms the **record route loaded**.

---

# 5. Detect Active Console Tab in DOM

Salesforce marks active tabs with:

```css
.slds-is-active
```

Example:

```javascript
const activeTab = page.locator('.workspace-tab.slds-is-active');

await expect(activeTab).toContainText('Test Account');
```

This confirms **the correct tab is selected**.

---

# 6. Wait for Lightning Data to Finish Loading

Lightning often shows loading spinners.

Wait until they disappear:

```javascript
await page.locator('.slds-spinner').waitFor({ state: 'hidden' });
```

Or:

```javascript
await page.waitForSelector('.slds-spinner', { state: 'detached' });
```

---

# 7. Combine Everything (Best Practice)

The most stable method combines **3 signals**:

1. Workspace API tab focus
    
2. URL change
    
3. Record component loaded
    

Example helper:

```javascript
async function waitForRecordLoad(page, recordId) {

  await page.waitForFunction(
    async (recordId) => {
      const workspaceAPI = document.querySelector('lightning-workspace-api');
      const tab = await workspaceAPI.getFocusedTabInfo();
      return tab.recordId === recordId;
    },
    recordId
  );

  await page.waitForURL(new RegExp(recordId));

  await page.waitForSelector('records-lwc-highlights-panel');

}
```

Usage:

```javascript
await waitForRecordLoad(page, '001XXXXXXXXXXXX');
```

---

# Example Real Test Flow

```javascript
test('Open account and verify tab', async ({ page }) => {

  const recordId = '001XXXXXXXXXXXX';

  await page.evaluate(async (recordId) => {
    const workspaceAPI = document.querySelector('lightning-workspace-api');

    await workspaceAPI.openTab({
      url: `/lightning/r/Account/${recordId}/view`,
      focus: true
    });
  }, recordId);

  await waitForRecordLoad(page, recordId);

  await expect(page.locator('records-lwc-highlights-panel'))
    .toContainText('Test Account');

});
```

# Common Mistake That Causes Flaky Tests

Many tests do:

```javascript
await page.click('text=Test Account');
await page.waitForTimeout(3000);
```

This fails because:

- Lightning rendering speed varies
    
- Tabs switch asynchronously
    
- DOM re-renders after navigation

# Enterprise Tip

A very powerful approach is to build a **Console Controller utility** in your framework:

```
consoleController.js

closeAllTabs()
getFocusedTab()
waitForTabFocus()
openRecordTab()
waitForRecordLoad()
```

All tests use this utility.

This reduces **Salesforce flakiness by ~80%**.
