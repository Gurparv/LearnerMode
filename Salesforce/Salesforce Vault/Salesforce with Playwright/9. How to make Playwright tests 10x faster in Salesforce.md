Salesforce Lightning apps are **notoriously slow for UI automation** because of heavy DOM, Aura/LWC rendering, and console tab loading. To make **Playwright tests ~10x faster**, high-performing Salesforce automation frameworks use a set of specific techniques.

Below are the **most impactful optimizations used in enterprise automation frameworks**.

# 1. Skip UI Login (Use `storageState`) 🚀

UI login takes **8–20 seconds per test**.

Instead login **once** and reuse the session.

### Global setup
```JavaScript
// global-setup.js
import { chromium } from '@playwright/test';

export default async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage();

  await page.goto('https://login.salesforce.com');

  await page.fill('#username', process.env.SF_USERNAME);
  await page.fill('#password', process.env.SF_PASSWORD);
  await page.click('#Login');

  await page.waitForURL('**/lightning/**');

  await page.context().storageState({ path: 'auth.json' });

  await browser.close();
};
```

### Playwright config
```Js
use: {  
storageState: 'auth.json'  
}
```

✅ Saves **~15 seconds per test**

# 2. Navigate Directly to Lightning URLs

Avoid clicking through menus.
Salesforce supports **deep links**.
Examples:
```Code
/lightning/o/Account/list
/lightning/o/Contact/list
/lightning/r/Account/{recordId}/view
```

Example
```js
await page.goto(`${baseUrl}/lightning/o/Account/list`);
```

Instead of:
```Code
App Launcher → Search Account → Click
```

✅ Saves **5–10 seconds**

# 3. Disable Animations & Lightning Delays

Lightning UI animations slow tests.

Inject CSS:
```Js
await page.addStyleTag({
  content: `
  * {
    transition-duration: 0s !important;
    animation-duration: 0s !important;
  }
`
});
```
✅ Speeds up UI actions significantly.


# 4. Avoid XPath (Use Lightning Stable Selectors)

XPath in Salesforce is **very slow** due to huge DOM.

❌ Bad
```js
page.locator('//span[text()="Accounts"]')
```

✅ Better
```js
page.getByRole('link', { name: 'Accounts' })
```

or
```js
page.locator('[title="Accounts"]')
```

# 5. Use API to Create Test Data Instead of UI

Creating records via UI can take **10–30 seconds**.
Instead use **Salesforce REST API**.
Example:
```js
import axios from 'axios';

await axios.post(
  `${instanceUrl}/services/data/v60.0/sobjects/Account`,
  { Name: 'Test Account' },
  { headers: { Authorization: `Bearer ${token}` } }
);
```

Then open directly:
```code
/lightning/r/Account/{id}/view
```

✅ Saves **20+ seconds per test**

# 6. Close Console Tabs Automatically

Too many tabs slow Lightning rendering.
```js
await page.evaluate(async () => {
  const workspaceAPI = document.querySelector('lightning-workspace-api');
  const tabs = await workspaceAPI.getAllTabInfo();

  for (const tab of tabs) {
    if (!tab.isPinned) {
      await workspaceAPI.closeTab({ tabId: tab.tabId });
    }
  }
});
```

# 7. Disable Unnecessary Resources

Block heavy resources.
```js
await context.route('**/*.{png,jpg,jpeg,gif,svg}', route => route.abort());
await context.route('**/analytics/**', route => route.abort());
```

Salesforce loads a lot of analytics scripts.

✅ Page load becomes faster.

# 8. Use `locator` Instead of `waitForSelector`

Playwright **auto-waits** with locators.

❌ Slow
```js
await page.waitForSelector('.slds-button');
await page.click('.slds-button');
```

✅ Faster
```js
await page.locator('.slds-button').click();
```

# 9. Run Tests in Parallel

Salesforce tests are often run sequentially but Playwright supports parallel.
```js
workers: 5
```

Example:
```
5 workers → 5x faster suite execution
```
Just ensure **test data isolation**.

# 10. Use Headless Mode in CI
Headless mode is faster.
```js
use: {
  headless: true
}
```

# 11. Avoid Global Navigation Reloads

Instead of:
```js
await page.goto('/lightning/page/home');
```

Prefer **tab navigation inside console**.

# 12. Use Lightning DOM Ready Signal

Lightning sometimes renders elements before data is ready.

Use:
```js
await page.waitForSelector('records-lwc-highlights-panel');
```

instead of random waits.
❌
```code
waitForTimeout(5000)
```
