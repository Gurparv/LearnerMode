5. How to Handle API testing in salesforce, and API verification of a record in salesforce to reduce UI heaviness and for quick automation testing.

```prompt
What is the best strategy for integrating API testing in Salesforce UI automation, and how can we verify records through Salesforce APIs instead of validating everything via the UI to reduce test execution time and UI dependency?
```

Your question targets a **very important Salesforce automation strategy**: combining **UI testing + API verification** to reduce UI dependency, make tests faster, and improve reliability.

In mature Salesforce automation frameworks, the rule is:

```text
Use UI for actions
Use API for verification
```

Instead of verifying everything through the UI.

Below is the **best-practice strategy used in enterprise Salesforce automation frameworks**.

---

# 1. Hybrid Testing Strategy (UI + API)

The recommended pattern is:

```text
UI Action → API Verification
```

Example workflow:

```text
1 Create Account through UI
2 Capture Record ID
3 Verify fields via Salesforce API
```

Instead of:

```text
Create Account UI
Navigate to list view
Search record
Open record
Verify fields in UI
```

Benefits:

|Benefit|Result|
|---|---|
|Faster tests|5–10x faster|
|Less UI flakiness|No list view loading|
|Better validation|Direct DB verification|

---

# 2. Capture the Record ID After UI Creation

When Salesforce creates a record, the **URL contains the record ID**.

Example URL:

```text
/lightning/r/Account/001XXXXXXXXXXXX/view
```

Extract it in Playwright:

```javascript
const url = page.url();
const recordId = url.split('/')[6];
```

Now you can verify the record using API.

---

# 3. Query Salesforce via REST API

Use **SOQL queries** through the REST API.

Example query:

```sql
SELECT Id, Name, Industry FROM Account WHERE Id='001XXXXXXXXXXXX'
```

Playwright example:

```javascript
async function queryRecord(request, soql) {

  const response = await request.get(
    `/services/data/v60.0/query?q=${encodeURIComponent(soql)}`
  );

  return await response.json();
}
```

Usage:

```javascript
const result = await queryRecord(
  request,
  `SELECT Name FROM Account WHERE Id='${recordId}'`
);

expect(result.records[0].Name).toBe("Test Account");
```

This confirms the record exists **without opening the UI again**.

# 4. Example UI + API Hybrid Test

```javascript
test('Create account and verify via API', async ({ page, request }) => {

  await page.goto('/lightning/o/Account/list');

  await page.getByRole('button', { name: 'New' }).click();

  await page.getByLabel('Account Name').fill('Playwright Account');

  await page.getByRole('button', { name: 'Save' }).click();

  const recordId = page.url().split('/')[6];

  const response = await request.get(
    `/services/data/v60.0/query?q=
     SELECT+Name+FROM+Account+WHERE+Id='${recordId}'`
  );

  const data = await response.json();

  expect(data.records[0].Name).toBe('Playwright Account');

});
```

Now the UI is used only for **creation**, not for validation.

# 5. Verify Trigger / Flow Results via API

Many Salesforce flows update other fields automatically.

Instead of checking UI:

```text
Open record
Scroll to field
Check updated value
```

Verify through API:

```javascript
const result = await queryRecord(
  request,
  `SELECT StageName FROM Opportunity WHERE Id='${opportunityId}'`
);

expect(result.records[0].StageName).toBe("Approved");
```

This validates **backend automation** instantly.

---

# 6. Validate Related Records

Example: Opportunity should create a Task via flow.

Query related records:

```sql
SELECT Id, Subject FROM Task WHERE WhatId='006XXXXXXXXXXXX'
```

Playwright example:

```javascript
const tasks = await queryRecord(
  request,
  `SELECT Subject FROM Task WHERE WhatId='${oppId}'`
);

expect(tasks.records.length).toBeGreaterThan(0);
```

This verifies **flow automation** without UI navigation.

---

# 7. Wait for Async Automation

Salesforce flows, triggers, or async jobs may take a few seconds.

Use polling:

```javascript
async function waitForRecord(request, soql) {

  for (let i = 0; i < 10; i++) {

    const result = await queryRecord(request, soql);

    if (result.records.length > 0) {
      return result;
    }

    await new Promise(r => setTimeout(r, 1000));
  }

  throw new Error("Record not found");
}
```

# 8. Create Reusable Salesforce API Service

Enterprise frameworks create a reusable API layer.

Example:

```text
services/
   salesforceApi.js
```

Example functions:

```javascript
queryRecord()
createRecord()
updateRecord()
deleteRecord()
waitForRecord()
```

Tests use these utilities instead of writing raw queries.

# 10. Best Practice Guidelines

|Rule|Why|
|---|---|
|Use UI only for actions|Mimics real user behavior|
|Use API for validation|Faster and stable|
|Use API for data setup|Avoid UI data creation|
|Use SOQL for verification|Direct backend validation|
