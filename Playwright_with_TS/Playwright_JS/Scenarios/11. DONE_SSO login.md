# 🔐 PART 1: Handling SSO Login in Playwright

SSO is tricky because:

- Redirects across domains
    
- MFA / CAPTCHA
    
- OAuth tokens / cookies
    
- Security restrictions (can’t automate easily)
    

---

## 🟢 Beginner Level

### ❌ Approach

- Automate login UI every time
    

```ts
test('login test', async ({ page }) => {
  await page.goto('https://app.com');
  await page.click('Login');
  await page.fill('#username', 'user');
  await page.fill('#password', 'pass');
  await page.click('Submit');
});
```

### 🚨 Problems

- Slow (every test logs in)
    
- Flaky (SSO redirects fail)
    
- CAPTCHA / MFA blocks
    
- Dependent tests
    

👉 **Mindset**: “Just make it work”

---

## 🟡 Intermediate Level

### ✅ Approach: Use `storageState`

Login once → reuse session

```ts
// global-setup.ts
import { chromium } from '@playwright/test';

export default async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage();

  await page.goto('https://app.com/login');
  // perform login manually or automated

  await page.context().storageState({ path: 'auth.json' });
  await browser.close();
};
```

```ts
// playwright.config.ts
use: {
  storageState: 'auth.json'
}
```

### 💡 Benefits

- No repeated login
    
- Faster tests
    
- More stable
    

### 🚨 Limitation

- Shared session → not isolated
    
- Token expiry issues
    

👉 **Mindset**: “Optimize speed”

## 🔵 Expert Level

### ✅ Approach: Programmatic Authentication (API-level)

Instead of UI → call auth APIs directly

```ts
test.beforeEach(async ({ request, context }) => {
  const response = await request.post('https://auth.com/api/login', {
    data: {
      username: 'testuser',
      password: 'password'
    }
  });

  const cookies = await context.cookies();
});
```

Or inject token:

```ts
await context.addCookies([{
  name: 'auth_token',
  value: token,
  domain: '.app.com',
  path: '/'
}]);
```

### 💡 Benefits

- Fastest
    
- No UI dependency
    
- Stable
    

### ⚠️ Needs

- Understanding OAuth / SAML / JWT
    
- Coordination with backend team
    

👉 **Mindset**: “Bypass UI, test system efficiently”

## 🔴 Elite (Production-Grade)

### ✅ Multi-Strategy SSO Handling

You don’t rely on ONE method. You build a **layered auth strategy**:

### 1. Auth Modes

- UI login → smoke tests
    
- API login → majority tests
    
- Pre-generated tokens → CI pipelines
    

---

### 2. Dynamic Session Management

```ts
if (tokenExpired) {
  regenerateToken();
}
```

---

### 3. Role-Based Auth

```ts
test.use({ storageState: 'admin.json' });
test.use({ storageState: 'user.json' });
```

---

### 4. Secret Management

- Use vaults (AWS Secrets Manager, Azure Key Vault)
    
- Never hardcode credentials
    

---

### 5. Parallel-safe Auth

Each worker gets its own session:

```ts
test.use({
  storageState: `auth-${workerIndex}.json`
});
```

---

### 🧠 Elite Insight

> “Login is NOT a test. It’s a dependency. Treat it like infrastructure.”
