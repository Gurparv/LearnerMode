### Network routing in playwright.
- Intercept network requests.
- Abort network requests.
- Modify network requests.
- Mock network requests.
- Simulate a slow network.

```Typescript
// Intercepting API requests
await page.route(
	"https://api.practicesoftwaretesting.com/products**",
	async(route) => {
	const response = await route.fetch();
	products = await response.json();
	route.continue();
	}
);
```
Here we are passing in the API for the products so we get a list of products back, and we're routing that through this page router. We then set the response to whatever comes back from the response. We can then go in and set that to a variable called products and then use products in our tests. With this , route.continue(), the last part is very important coz this says go ahead and complete the network request, so we've intercepted it.

```Typescript
// Abort network requests
await page.route(
"https://api.practicesoftwaretesting.com/categories/tree**",
(route) => route.abort()
);
```
Another option is aborting it, You still call page.route, but instead of fulfilling it or continuing with it, you are actually telling the route to abort the request, you're actually telling the route to abort the request, any other requests that match this pattern that we're passing it. 
This is very helpful specifically in one situation where I've used this was to block Google ads on a certain site. There was a site i was testing and a Google ad would always pop up over the login button, which caused problems when you are trying to create a login test, and aborting that request allowed me to not load any of the Google ads and not have any issues with the test and flakeyness. That can also be used to block out images or anything that you need to do when making a visual test as well.

```Typescript
// Modify network requests
await page.route(
	"https://api.practicesoftwaretesting.com/products**",
	async (route) => {
		const response = await route.fetch();
		const json = await response.json();
		json.data[0]["name"] = "Mocked Product";
		json.data[0]["price"] = 100000.01;
		json.data[0]["in_stock"] = false;
		
		await route.fulfill({response, json});
	}
);
```
this is very similar to intercepting a API request except here we are going to modify it, so what gets sent back , what gets fulfilled, is actually different.

```Typescript
await page.routeFromHAR(".hars/product.har", {
url: "https://api.practicesoftwaretesting.com/products**",
update: false,
});
```
Mocking API request, The mocking happens using this routeFromHAR, or theres a routeFromJSON as well, but what this method does is you add it and you set the update parameter to true. So anytime it is set to true, when the test runs it saves the HAR file to your test directory wherever you specify, and you commit that HAR file to the repo, so you basically have this mocked network request, that can always be returned. Of course, when you commit this, you're going to change your update back to false, and now you have a test that's always mocking whatever was saved to the repo, This is very helpful when working with third-party APIs that you dont have any control over.

```Typescript
await page.route(
	"https://api.practicesoftwaretesting.com/products**",
	async (route) =>{
		await page.waitForTimeout(4000);
		await route.continue();
	}
);
```
In this scenario, we're going to put this route in place, and before we continue with the route, we basically have a four-second timeout, you know using Playwright's built-in waitForTimeout function. This allows you to test and simulate what its like working with slow network requests within your test case.

## Intercepting HTTP requests with Playwright

```Typescript title="home.spec.ts"
test("validate product data is visible in UI from API", async ({ page }) => {
    let products: any;
    await test.step("intercept /products", async () => {
      await page.route(
        "https://api.practicesoftwaretesting.com/products**",
        async (route) => {
          const response = await route.fetch();
          products = await response.json();
          route.continue();
        }
      );
    });

    await page.goto("/");

    await expect(page.locator(".skeleton").first()).not.toBeVisible();
    const productGrid = page.locator(".col-md-9");

    for (const product of products.data) {
      await expect(productGrid).toContainText(product.name);
      await expect(productGrid).toContainText(product.price.toString());
    }
  });
});
```

### Mocking HTTP responses with Playwright.
```Typescript title="home.spec.ts"
test("validate product data is visible from modified API", async ({ page }) => {
  await test.step("overwrite /products", async () => {
    await page.route(
      "https://api.practicesoftwaretesting.com/products**",
      async (route) => {
        const response = await route.fetch();
        const json = await response.json();
        json.data[0]["name"] = "Mocked Product";
        json.data[0]["price"] = 100000.01;
        json.data[0]["in_stock"] = false;

        await route.fulfill({ response, json });
      }
    );
  });
  await page.goto("/");

  const productGrid = page.locator(".col-md-9");
  await expect(productGrid.getByRole("link").first()).toContainText(
    "Mocked Product"
  );
  await expect(productGrid.getByRole("link").first()).toContainText(
    "100000.01"
  );
  await expect(productGrid.getByRole("link").first()).toContainText(
    "Out of stock"
  );
});

test("validate product data is loaded from har file", async ({ page }) => {

      url: "https://api.practicesoftwaretesting.com/products**",
      update: false,
    });
  });
  await page.goto("/");
  const productGrid = page.locator(".col-md-9");
  await expect(productGrid).toContainText("Happy Path Pliers");
  await expect(productGrid).toContainText("1.99");
});
```

### Injecting Javascript in Playwright

In some situations out of the box playwright methods werent able to solve my problems. Playwright 'evaluate' functionality allows you to inject and run any Javascript directly on any webpage. The page evaluate functionality is typically my last resort.

Using `page.evaluate` in Playwright lets you run custom JavaScript directly inside the webpage, which is powerful when standard Playwright methods can't solve a problem. It's often a last resort but gives you flexibility to handle complex or unique scenarios—like checking for inputs without labels or detecting broken images by executing JavaScript tailored to your page's structure. This approach helps you create more precise and thorough tests when built-in Playwright functions aren't enough.

With this functionality the possibilities are literally endless, I found any of the AI LLM have great ideas of Javascript should i execute on a page to help identify future errors.

```Typescript title="home.spec.ts"
 test("check for broken images", async ({ page }) => {
    // await page.goto("https://with-bugs.practicesoftwaretesting.com/");

    const brokenImages = await page.evaluate(() => {
      return Array.from(document.querySelectorAll("img"))
        .filter((img) => img.naturalWidth === 0 || img.naturalHeight === 0)
        .map((img) => img.src);
    });
    expect(
      brokenImages.length,
      `Broken Images: ${brokenImages.toString()}`
    ).toBe(0);
  });
});
```
