One Pro of Playwright is that playwright gives you these browsers out of the box and you are able to use them with very little configuration.

One of the drawbacks when using Playwright is you're locked into a specific version of Chromium, Firefox or webkit. This version that is packaged with each version released with Playwright. What this means is you will be limited to whatever browser version is bundled with playwright.

'projects' property in playwright configuration also can be used to set up data, to segregate tests, or to override certain browser configurations on a per-project basis.

## Explore VS Code extension.
Here are some key takeaways from the video on using the Playwright Test VS Code extension:  
- The extension allows you to easily load, filter, run, and debug Playwright tests directly within VS Code.
- Debugging features include setting breakpoints, stepping through code line-by-line, and inspecting elements in the browser as tests run.
- Continuous test running automatically reruns tests on file save, which is great for quick feedback during development.
- You can show the browser during test runs and use the trace viewer to analyze test execution.
- The extension also supports recording new tests by interacting with the browser, which helps generate test code quickly.

  
These tools can significantly streamline writing and maintaining automated tests in your workflow.

## Playwight UI Mode.
Here are the key takeaways from the video on using Playwright UI mode:  
- Playwright UI mode allows you to run and update tests locally with immediate feedback, making test development faster and more interactive.
- You can filter tests by tags, view detailed test execution steps, and see the corresponding source code to understand what actions are performed.
- The UI provides rich debugging tools including logs, error messages, network requests, and a timeline to analyze test behavior and failures.
- When a test fails, you can quickly identify the issue, fix the code, and see the test rerun and pass instantly.

  
This tool is especially helpful for efficiently building and maintaining automated tests with clear visibility into test execution and errors.
