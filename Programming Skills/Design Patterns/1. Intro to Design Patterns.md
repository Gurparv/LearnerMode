- They are basically common `patterns`  in programming
- Design patterns are aimed towards Object-Oriented Programming languages.
- Creating Object-oriented software is hard, and designing reusable object-oriented software is even harder. So `Design Patterns` address this issue and give us standard.
- Design pattens focus more on reuse of recurring architectural design themes, while frameworks focus on detailed design and implementation.
- Patterns identify and specify abstractions that are above the level of single class and instances, or components.
- Design patterns are not just about the design of objects
	- also about interaction between objects.
	- include strategies for object inheritance and containment.
- Each pattern describes a problem which occurs over and over again in our environment and then describes the core of the solution to that problem , in such a way that u can use this solution a million times over, without ever doing it the same way twice.
- Book -> ==Design patterns - elements of Reusable Object-Oriented Software==

## 4 Essential Elements
- a design pattern has four essential elements.
1. Pattern Name 
2. Problem 
3. Solution
4. Consequences

---

## Types of Design Patterns

By Purpose
1. Creational - concern the process of object creation.
2. Structural - deal with the composition of classes or objects.
3. Behavioral - Characterize the ways in which classes or objects interact and distribute responsibility.

By Scope
1. Class Patterns -> deal with classes and subclasses
2. Object Patterns -> deal with object relationships

---

## Creational patterns
1. Creational design patterns provide a way to create objects.
2. Simplest way to create in java `Fred = new Fred();`
3. Creational Design patterns abstracts the instantiation process
	1. The creation logic is hidden.
	2. encapsulates knowledge about which concrete classes the system uses
	3. programmer may call a method or use another object , rather than instantiating objects directly using the new operator.


## Structural Patterns
1. Describes how classes and objects can be combined to form larger structures
	- utilizes inheritance to compose interfaces or implementations.
	- structural object patterns describe ways to assemble objects.
2. The Composite design pattern
	- describes how to build a class hierarchy made up of classes for kinds of objects.
3. The Proxy design pattern acts as a placeholder for another objects.

## Behaviour Patterns
1. These design patterns are specifically concerned with communication between objects
	- characterize complex control flow that is difficult to follow at run-time
	- shift the focus away from flow of control to let you concentrate just on the way objects are interconnected.
