## Introduction

Clean code -> Code which is `easy to read` and `understand`
Its characteristics
1. Should be `readable and meaningful`
2. Should `reduce cognitive load`
3. Should be `concise` and "to the point"
4. Should `avoid unintuitive names, complex nesting` and `big code blocks`
5. Should follow `common best practices` and `patters`
6. Should be `fun` to `write` and `to maintain`

Key Pain Points that make code 'bad'.
- Naming of `Variables`, `Functions`, and `Classes`
- Structure and Comments - Code formatting , Good & Bad Comments
- Functions - Length and Parameters
- Conditionals & Error Handling - Deep Nesting, Missing Error Handling
- Classes and Data Structures - Missing Distinction and Bloated Classes.

The Solutions to these KeyPoints are
1. Rules and Concept
2. Patterns and Principles
3. Test-Driven Development

- How to **write** the code
