## What makes a Function?
- Function body -> The length of function body matters.
- Function Calling -> calling the function should be readable. That is number and order of arguments matter.
- So both of them should be readable.

### Functions Calling
#### Number of Function/Method Parameters
In Best to Worst Order 
- None
	- `user.save()`
- One parameter 
	- `log(message)`
- Two parameters
	- `add(5, 10)`
- Three parameters
	- `calc(5, 10, 'add')`
- More parameters
	- `coords(10, 3, 9, 12)`


### Functions Body
- Functions body should be small.
- Functions should do exactly `one thing`.
	- For finding `one thing` we use combination of `Different Operations` and `levels of Abstraction that is High and Low level of abstraction in a function`

#### Functions & Abstraction
Functions should do work that's `one level of abstraction below their name.`
eg
```Javascript 
// Good 
function emailIsValid(email){
	return email.includes('@');
}

// Bad
function saveUser(email){
	if(email.includes('@')) {...}
	// ...
}
```

#### Keeping functions short
Rule of Thumb
1. Extract code that works on the same functionality eg
```Javascript
user.setAge(31)
user.setName('Max')
// can be converted to 
user.update ({age:31, name: 'Max'})
```

2. Extract code that requires more interpretation than the surrounding code.
```Javascript
if (!email.includes('@')) {...}
	saveNewUser(email)
```

### Splitting functions to stay DRY -> Dont Repeat Yourself
Dont write the same code or same type of code more than once.
##### Signs of code which 'is not DRY'
- you are copy/pasting code.
- You need to apply the same change to multiple places in your codebase.
##### Dont Overdo it - Avoid Useless Extractions
Make reasonable decisions and dont split if
- you are just renaming the operation.
- finding the new function will take longer than reading the extracted code.
- can't produce a reasonable name for the extracted function.

### Have 'Pure Functions'
Pure functions are the functions which produces the same input always yields the same output.
Pure functions dont have side effects.
- Side effect is an operation which doesnt **not** just act on function inputs and change the function output but which instead **changes the overall system/ program state**

```Javascript
// Pure Function
function createUser (email, password){
	const user = new User (email, password);
	return user;
}

// Not a Pure function
function generateId(username) {
```

#### Handling Side Effects
Your function should **not** have any **unexpected side effects.**
If you have/ need a side effect
1. choose a function name which implies it
2. Move the side effect into another function / place
