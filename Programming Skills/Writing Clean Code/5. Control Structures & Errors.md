### Introducing Guards and Fail Fast.
Super efficient way to remove unnecessary nesting.
```Javascript
//Program without guard
if (email.includes('@')) {
	// do stuff
}

// program with guard (Fails Fast)
if (!email.includes('@')) {
	return;
}
// do stuff
```

Example 2
```Javascript
if(user.active) {
	if (user.hasPurchases()) {
		// do stuff
	}
}

if (!user.active){
	return; // Guard
}
if (!user.hasPurchases()){
	return; // Guard 2
}
// do stuff

```

### Embrace Errors And Error handling
Throwing + Handling errors can replace if statements and lead to more focused functions.
Simple Rule: If something is an error -> Make it an error.
eg
```Javascript
//Bad
if (!isEmail){
	return {code: 422, message: 'Invalid input'};
}

//Good : this throws error but you need to handle errors using try/catch
if (!isEmail){
	const error = new Error ('Invalid input');
	error.code = 422;
	throw error;
}

```
