## How AI Agents work with LLMs and Prompts

### It all starts with prompting
1. An LLM receives an input and predicts what comes next
	- It is stateless; the call to an LLM like GPT depends only on the input.
2. The prompt can be made up of several parts
	- The system prompt gives overall context including role the LLM is playing.
	- The user prompt gives the message that the LLM is responding to 
	- The whole conversation so far can be included.
3. The key is to finesse the input and interpret the output
	- From "prompt engineering" to "context engineering".

## Day 4

### What is JSON?
A simple way to represent structured data that's easy to read.
1. Key/Value pairs - Of Strings, numbers, booleans...
2. An Object - A bunch of key/value pairs, also called a dictionary.
3. An Array - An ordered list
4. Nesting - Values can themselves be objects and lists.

```Json
// Key-value pairs
"name" : "Alice"
"age" : 30
"middle_name" : null
"is_student" : false

// An Object -> A group of key/value pairs in {} representing a thing.
{
	"name" : "Alice",
	"age" : 30,
	"is_student" : false,
	"middle_name" : null
}

// An Array -> A list of things in [..]
["apples", "bananas", "oranges"]
[1, 1, 2, 3, 5]
[1, true, "bananas"]

//Nesting -> List of objects OR
[
	{
		"name" : "Alice",
		"age" : 30
	},
	{
		"name" : "Bob",
		"age" : 35
	}
]
// Nesting -> An object can contain an object
{"street" : "123 street", "city" : "NewYork", "country":"US"} // object 1

{
	"name": "Alice",
	"age" : 30,
	"address" : {"street" : "123 street", "city" : "NewYork", "country":"US"}
}

```

### Expressions in n8n
Determine something on the fly, rather than using a fixed value.

Toggle between "expression" and "fixed"

```JSON
		"street" : "1 Main Street",
		"city" : "New York",

	}

```

```Bash

{{ $json.address.city }}
```
